# Internal Linking Map

Use this file to keep linking between homepage, services, child pages, work, tools, and contact intentional.

## Core rule
Internal links should reinforce the layered content system:
- homepage = orient
- services index = route
- child-service pages = clarify and convert
- work = prove
- tools = diagnose / give value
- contact = close the loop

Do not link randomly just because pages are topically related.

### Child-service pages
Each child-service page should link to:
1. `/contact?intent=service`
2. one primary proof page
3. one related sibling page if natural
4. a tool only when it meaningfully supports the service

### Work pages
Each work page should ideally link to:
- the service page it best proves
- `/contact?intent=work`
- possibly one related tool if naturally relevant

### Tools pages
Each tool or tool index should link to:
- one main service CTA
- one supporting proof page when helpful
- `/contact?intent=tool` where the user has already engaged with the tool

## Linking rules by page type

### Homepage rules
- do not link to too many child-service pages directly from the hero
- do not make the homepage a substitute for `/services`
- keep service detail routing mostly inside `/services`

### Services index rules
- every cluster should point deeper
- cluster-level proof should support, not dominate
- avoid linking to 5+ proof pages inside one cluster block

### Child-service page rules
- one primary proof CTA
- one primary contact CTA
- one optional related sibling service
- one optional tool only if it helps clarify the offer

### Work page rules
- link back to the service page that best matches the mechanism of the result
- not every work page needs multiple service links
- one strong service match is better than three weak ones

### Tool page rules
- tool CTAs should not scatter users into too many options
- prefer one service CTA and one next-step route

## Intent-aware contact routing
Use these defaults:
- broad homepage CTA Ôćĺ `/contact?intent=unsure`
- service page CTA Ôćĺ `/contact?intent=service`
- work page CTA Ôćĺ `/contact?intent=work`
- tool page CTA Ôćĺ `/contact?intent=tool`

## Anti-patterns
Avoid:
- linking every service page to the same 3 proof pages without reason
- adding tool links where they distract from a clear service CTA
- stacking multiple contact CTAs in one section
- sending users back up the information ladder unnecessarily

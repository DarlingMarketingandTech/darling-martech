## Rules for assigning proof

### 1. One primary proof per page
Every service page should have one clearly dominant case study or system proof.

### 2. Supporting proof should clarify, not clutter
Use supporting proof only when it adds range, industry relevance, or a stronger mechanism story.

### 3. Proof should match the service claim
Do not attach visually similar projects when the actual operational or strategic proof is weak.

### 4. Priority 1 pages should use the strongest, cleanest proof first
That usually means:
- Graston Technique
- Pike Medical Consultants
- Graston Growth Engine
- Russell Painting
- 317 BBQ

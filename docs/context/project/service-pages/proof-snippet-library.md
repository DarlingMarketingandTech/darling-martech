# Proof Snippet Library

Use this file when writing homepage proof, service-page proof blocks, service index cluster proof, and CTA-adjacent credibility snippets.

These are reusable proof fragments, not full case-study summaries.

## Usage rules
- use one dominant metric/snippet at a time
- keep snippets tied to the page’s service claim
- do not stack too many proof statements in one block
- use service-relevant framing, not generic client-name repetition

## Proof block format suggestion
When writing a proof block, structure it like this:
1. service-relevant proof title
2. one short summary line
3. one dominant metric
4. one line tying it back to the canonical work page

Example:
- **Healthcare website rebuild and trust architecture**
- Clearer structure, stronger trust, and a better path from discovery to action.
- **45% patient growth over 3 years.**
- From the Pike Medical Consultants case study.

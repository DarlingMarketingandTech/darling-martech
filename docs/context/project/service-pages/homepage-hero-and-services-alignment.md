# Homepage Hero + Services Alignment Notes

Use this file when refining the homepage hero and its relationship to the services section.

## Current strategic direction

The homepage should not repeat the founder-positioning story in multiple sections.
The hero should carry the core positioning load.
The services section should then translate that promise into problem clusters.

## Core rule

### Hero job
The hero should answer:
1. who this is for
2. what is broken
3. what Darling MarTech fixes
4. why the strategy + systems model matters
5. what the best next step is

### Services section job
The services section should answer:
1. what kinds of problems are solved here
2. which broad cluster the visitor belongs in
3. where to click next for more specificity

## What to avoid
- repeating the full "both sides / one person / no hand-offs" story in a second homepage section
- making the services section repeat the exact hero claim without adding structure
- turning the hero into a long biography block

## Better flow

Hero
→ problem + promise + model

Services
→ problem clusters + child-page directions

Tool
→ immediate value

Proof
→ evidence

Testimonials
→ trust

Contact CTA
→ next step

## Hero copy guidance

### Keep
- founder-led strategic + technical differentiator
- calm premium tone
- strong left-aligned hierarchy

### Strengthen
- buyer problem framing
- clarity on what kinds of businesses this is for
- clearer primary and secondary CTA logic

### Reduce
- repetitive founder-story language
- too much resume-like framing
- vague or high-context wording that cold visitors have to decode

## Services intro guidance

The services section should start from:
- disconnected systems
- weak conversion
- unclear messaging
- low visibility
- manual follow-up
- weak measurement

Then map those problems to the four clusters.

## Use case

Use this file before making homepage hero changes or rewriting the services intro so both sections work together instead of competing with each other.

# Service Page Implementation Tracker

Use this file to move from planning docs to actual implementation in a controlled order.

## Status legend
- `planned` — brief exists, page not started
- `drafting` — implementation copy or route work in progress
- `built` — route/content live in repo
- `needs-proof-review` — page exists, but proof/supporting links need work
- `needs-copy-polish` — page exists, structure is there, copy needs tightening

## Page build checklist

For every implementation PR, verify:
- route exists
- H1 is plain-English and specific
- opener uses Layer 1 language first
- page includes Signs You Need It section
- page includes What It Usually Includes section
- technical scope is lower on the page
- proof block includes one primary proof item
- CTA points to `/contact?intent=service`
- internal links point to related proof and sibling service pages
- metadata/title/description are set
